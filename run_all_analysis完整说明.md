# run_all_analysis.py - 完整分析流程说明

## 📋 概述

`run_all_analysis.py` 是广东省招聘数据分析项目的**主入口脚本**，它按照预定的顺序执行所有分析模块，从原始数据到最终的Excel汇总报告，实现一键式完整分析。

---

## 🔄 完整工作流程（6个步骤）

### 步骤0：数据整合 - 添加职业类别字段

**模块**：`src.preprocessing.integrate_occupation.DataIntegrator`

**输入**：
- `output/split_out/*_1%.csv` - 原始样本数据（1%）

**处理逻辑**：
1. 读取原始CSV数据
2. 使用**职业解析器**（OccupationParser）解析岗位名称
   - 算法：**最长后缀匹配**
   - 词典：270个职业核心词
   - 输出：occupation_core（职业）、occupation_category（职业类别）
3. 添加职业字段到数据中
4. 保存整合后的数据

**输出**：
- `output/integrated/前程无忧_整合_1%.csv`
- `output/integrated/智联招聘_整合_1%.csv`
- `output/integrated/猎聘网_整合_1%.csv`

**为什么需要**：
- 原始数据只有"岗位名称"，需要提取结构化的"职业"和"职业类别"
- 为后续的职业维度分析提供基础数据
- 使用规则匹配而非机器学习，保证98%的准确率和完全可解释性

---

### 步骤1：职业类别薪资分析

**模块**：`src.analysis.occupation_salary_analysis.OccupationSalaryAnalyzer`

**输入**：
- `output/integrated/*_整合_*.csv` - 整合后的数据

**处理逻辑**：

#### 1.1 薪资解析
- **算法**：正则表达式匹配
- **支持格式**：
  - "10-15万" → 最低10万，最高15万
  - "8000-12000元" → 最低8000，最高12000
  - "15-20k" → 最低15000，最高20000
- **计算**：平均薪资 = (最低薪资 + 最高薪资) / 2

#### 1.2 职业类别月度薪资分析
- **分组**：按 occupation_category（职业类别）+ publish_month（月度）
- **聚合**：计算平均薪资、岗位数量
- **过滤**：只保留岗位数量>=5的数据点
- **输出**：职业类别月度薪资数据.csv（429行）

#### 1.3 职业月度薪资分析（主口径）
- **分组**：按 occupation_core（职业）+ publish_month（月度）
- **范围**：Top 20职业
- **聚合**：计算平均薪资、岗位数量
- **过滤**：只保留岗位数量>=5的数据点
- **输出**：职业月度薪资数据.csv（577行）

#### 1.4 学历×职业类别交叉分析
- **分组**：按 occupation_category（职业类别）+ 学历
- **学历提取**：从"学历要求"字段提取（博士、硕士、本科、大专）
- **聚合**：计算平均薪资、岗位数量
- **过滤**：只保留岗位数量>=5的数据点
- **输出**：学历职业类别薪资数据.csv（74行）

#### 1.5 学历×职业交叉分析（主口径）
- **分组**：按 occupation_core（职业）+ 学历
- **范围**：Top 15职业
- **聚合**：计算平均薪资、岗位数量
- **过滤**：只保留岗位数量>=3的数据点
- **输出**：学历职业薪资数据.csv（39行）

**输出文件**：
- `output/reports/职业类别薪资分析报告.txt` - 文本报告
- `output/reports/职业类别薪资分析图.html` - 交互式图表
- `output/reports/职业类别月度薪资数据.csv` - 数据表
- `output/reports/职业月度薪资数据.csv` - 数据表
- `output/reports/学历职业类别薪资数据.csv` - 数据表
- `output/reports/学历职业薪资数据.csv` - 数据表

**为什么需要**：
- **职业维度**是招聘分析的核心维度
- **月度趋势**揭示薪资变化和市场动态
- **学历交叉**分析学历对薪资的影响
- **双口径**（职业类别+职业）提供粗细粒度分析

### 步骤2：行业景气度分析

**模块**：`src.analysis.industry_trend_analysis.IndustryTrendAnalyzer`

**输入**：
- `output/integrated/*_整合_*.csv` - 整合后的数据

**处理逻辑**：

#### 2.1 行业月度招聘量分析
- **分组**：按 industry_clean（行业）+ publish_month（月度）
- **聚合**：计算岗位数量
- **过滤**：只保留岗位数量>=5的数据点
- **输出**：行业月度数据.csv（2,417行）

#### 2.2 城市×行业月度分析
- **分组**：按 city_clean（城市）+ industry_clean（行业）+ publish_month（月度）
- **聚合**：计算岗位数量
- **过滤**：只保留岗位数量>=3的数据点
- **输出**：城市行业月度数据.csv（5,158行）

#### 2.3 行业景气度计算
- **算法**：
  - 计算每个行业的月度招聘量
  - 计算增长率：(本月 - 上月) / 上月
  - 识别热门行业（招聘量Top 10）
  - 识别增长最快行业（增长率Top 10）

**输出文件**：
- `output/reports/行业景气度分析报告.txt` - 文本报告
- `output/reports/行业景气度分析图.html` - 交互式图表
- `output/reports/行业月度数据.csv` - 数据表
- `output/reports/城市行业月度数据.csv` - 数据表

**为什么需要**：
- **行业维度**反映经济结构和产业趋势
- **景气度指标**帮助识别热门行业和就业机会
- **城市×行业**揭示地域产业特点
- **月度趋势**捕捉行业周期和季节性

## 🎯 核心算法总结

### 1. 职业提取算法：最长后缀匹配
```
输入："Java开发工程师"
步骤：
  1. 遍历核心词词典（按优先级）
  2. 匹配后缀："开发工程师" ✓
  3. 提取修饰词："Java"
输出：
  - occupation_core: "开发工程师"
  - occupation_category: "技术类"
```

### 2. 薪资解析算法：正则表达式
```
输入："10-15万/年"
步骤：
  1. 匹配模式：(\d+)-(\d+)万
  2. 提取：最低10万，最高15万
  3. 转换：年薪 → 月薪（÷12）
输出：
  - 最低薪资: 8333元/月
  - 最高薪资: 12500元/月
  - 平均薪资: 10417元/月
```

### 3. 学历标准化算法：关键词匹配
```
输入："本科及以上"
步骤：
  1. 遍历学历列表：[博士, 硕士, 本科, 大专]
  2. 匹配："本科" ✓
输出：
  - 学历: "本科"
```

### 4. 聚合统计算法：GroupBy + Agg
```
输入：DataFrame with [职业, 月度, 薪资]
步骤：
  1. 按职业+月度分组
  2. 计算平均薪资：mean()
  3. 计算岗位数量：count()
  4. 过滤：岗位数量 >= 阈值
输出：
  - 职业月度薪资表
```
