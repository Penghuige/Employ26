# V3技能分类器 - 自我迭代和自动验收方案

## 问题分析

### 原始置信度的问题

从 `v3_discovered_skills.txt` 的10个新技能来看：

**靠谱的（6个，60%准确率）**：
- ✓ Mysql (0.967) - 数据库
- ✓ ES6 (0.956) - JavaScript标准
- ✓ JAVA (0.896) - 编程语言
- ✓ webpack (0.893) - 前端打包工具
- ✓ Solidworks (0.889) - 3D CAD软件
- ✓ IATF16949 (0.873) - 汽车行业质量标准

**有问题的（4个，40%误报率）**：
- ✗ Server (0.901) - 太泛化，不是具体技能
- ✗ UART (0.853) - 硬件通信协议，过于专业/小众
- ✗ EDA (0.852) - 缩写歧义（可能是电子设计自动化，也可能是其他）
- ✗ NET (0.851) - 太泛化，可能是.NET但不确定

**结论**：单纯依赖置信度，准确率只有60%，需要多维度验证。

---

## 解决方案：三层自动验收机制

### 1. 多维度验证器（SkillValidator）

**验证维度**：

1. **基础过滤**
   - 长度检查：2-30字符
   - 黑名单模式：排除泛化词（能力、经验、Server、System等）

2. **技能特征加分**（+0.05分/特征）
   - 驼峰命名：JavaScript, TypeScript
   - 全大写缩写：SQL, AWS, GCP
   - 包含数字：C++, Vue3, Python3
   - 包含连字符：Node.js, Vue.js, C#

3. **文档频率检查**
   - 低频（<5）：-0.1分
   - 高频（>100）：+0.05分

4. **类别一致性**
   - 词汇包含类别关键词：+0.05分

**验证结果**（10个技能）：
- ✅ 高置信度（自动通过）：7个（70%）
- ⚠️ 需要人工审核：2个（20%）
- ❌ 自动拒绝：1个（10%）

### 2. 三级分类策略

| 类别 | 验证分数 | 处理方式 | 示例 |
|------|---------|---------|------|
| 高置信度 | ≥0.90 | 自动通过，加入训练集 | Mysql (1.017), ES6 (1.006), JAVA (0.946) |
| 需审核 | 0.80-0.90 | 人工审核 | webpack (0.893), Solidworks (0.889) |
| 自动拒绝 | <0.80 | 直接拒绝 | Server (0.000) |

### 3. 主动学习（ActiveLearner）

**策略**：优先标注不确定性高的样本

- **不确定性**：距离决策边界（0.8）的距离
- **重要性**：文档频率
- **综合分数**：不确定性×0.7 + 重要性×0.3

**选择的5个样本**（按不确定性排序）：
1. UART (0.853) - 边界样本
2. EDA (0.852) - 边界样本
3. Solidworks (0.889) - 边界样本
4. IATF16949 (0.873) - 边界样本
5. JAVA (0.896) - 边界样本
